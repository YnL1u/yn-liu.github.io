## 初始症状

部署日志中出现：

- Build failed
- Missing dependency
- 配置文件错误

网站无法成功生成。

## 排查过程

### 1. 查看完整构建日志

在 Vercel 控制台中查看完整 Build Log，确认错误发生在构建阶段。

### 2. 检查依赖与配置

重点检查：

- package.json
- 构建命令
- Node.js 版本

发现部分依赖未正确安装。

### 3. 检查环境变量

部分 API 调用依赖环境变量，如果未配置也会导致构建失败。

### 4. 本地构建测试

在本地执行：npm run build

如果本地构建失败，说明问题来自代码或依赖。

## 解决方案

最终通过以下方式解决：

- 修正依赖配置
- 确认构建命令
- 重新部署项目

部署成功后，网站可以正常访问。

## 总结

这次排查让我更加理解：

- 部署平台的构建流程
- 日志的重要性
- 本地测试的重要性

遇到部署问题时，**第一步一定是看完整日志，而不是盲目修改配置**。
